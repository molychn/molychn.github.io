JavaScript中的循环与遍历

发表于 2019-04-25 | 更新于: 2019-04-28 | 分类于 计算机科学

之前业务代码中遍历循环一把梭哈for循环就是干，纠结于写出优雅的代码，励志从“搬砖的”码农进化到“和水泥”的（水泥用于在砖缝之间将各个砖头连接起来，自认为写出好的代码也是将自身的各项编码实力连接起来），先总结一波关于js中的循环与遍历，以后有新的再补充。

原生JS循环与遍历

while

只要condition满足条件则会一直执行while代码块里的statement语句，所以在块内会进行状态的改变，从而终止循环

do while

相比较while循环，do…while循环需要先执行do代码块中的语句一次而后再开始循环

for

for循环中主要根据中间的条件语句实现循环，每次循环执行后面代码块的语句，而后有第三个参数控制整个循环的状态，一般是自增以达到突破条件从而终止循环。在循环过程中，continue，return，break都会终止循环或跳过当前循环进入下一次。关于for循环会经常联系作用域出面试题：

因为setTimeout的原因导致了作用域的改变（遇setTimeout宏任务会先暂停执行，等上一次的宏任务与微任务执行完成后执行），从而输出的i值全部都是全局作用域的值5，而且这里会得到5的原因在于for循环执行完毕后依旧会执行第三个语句参数，即i++。解题方法有几种，一是替换i的定义从var到let，因为let是自形成一个代码块；或者是实现一个立即执行函数，同时将i传递进去，每次传入的i值得以保存在该函数的作用域内。

关于传递i值，如果直接在setTimeout上传递i，则默认会因为循环结束后获取不到全局的i变量从而得到undefined的结果

for…in…

forin循环直接针对对象进行遍历，它的结果根据key->value格式输出。forin的遍历不仅将自身的可枚举属性遍历出来，还包括其原型链上的可枚举属性，因此对于自身可枚举的属性可以搭配hasOwnProperty方法食用更佳。

for…of…

for…of 语句创建一个循环来迭代可迭代的对象。在 ES6 中引入的 for…of 循环，以替代 for…in 和 forEach() ，并支持新的迭代协议。for…of 允许你遍历 Arrays（数组）, Strings（字符串）, Maps（映射）, Sets（集合）等可迭代的数据结构等。
ES6中的Generators(生成器)所生成的可迭代函数也可以被遍历，而普通的对象却不可迭代。

JavaScript 提供了四种已知的终止循环执行的方法：break、continue、return 和 throw。

Object.keys

相比forin循环，Object.keys的遍历会返回一个数组，该数组成员都是该对象可枚举的属性名。而Object.getOwnPropertyNames可以返回自身对象所有属性名，包括不可枚举。

数组遍历

遍历数组元素并执行一些操作，使用forEach或map都可以做到，区别在于map除了执行语句外，可以return回一个经过执行语句处理过的数组，而forEach即使return了也没用。

看方法名叫filter就知道它是数组过滤的方法，可以返回数组中符合条件的元素，生成新数组，并不会修改原数组。

some遍历是只要在遍历过程中发现存在符合条件的结果就返回true，并不会再往下遍历；而every则与之相反，需要对整个数组内的元素进行遍历才能得到结果;find的过程与some类似，只要找到第一个符合条件的就返回，但相比some返回true，find会返回符合条件的那个元素，而如果使用findIndex方法的话相比find返回元素，它会返回元素在原数组中的索引值。

reduce的参数比较多，方法可以传递两个参数，第一个为函数回调，第二个为执行reduce第一次预先传递进去的值。在回调函数中，第一个表示上次结果中累计的值，第二个位当前值，第三为当前索引，第四为原数组。
