Vue源码学习11：VirtualDOM与diff

发表于 2019-07-24 | 更新于: 2019-07-24 | 分类于 计算机科学

DOM渲染

一个真实的DOM节点其携带的默认属性是很多的，而在框架内数据双向绑定的情况下，每改变数据都会使得节点重新渲染，而如果让页面每次都整体进行渲染，在大型页面项目中会严重拖慢渲染的效率，因此需要Virtual DOM用于保存各个节点的信息，通过diff算法进行比对，实现特定节点更新渲染。

因此根据真实DOM生成Virtual DOM，当Virtual DOM某节点数据发生改变则生成新Vnode与oldVnode作对比，结果不一样的情况下直接修改真实DOM节点。

vnode patch过程

在Vue中，通过初始化会进行第一次页面的渲染，它大体的步骤如下面7个方法(函数)步骤：

  1. $mount
  2. mountComponent
  3. vm._update
  4. vm.patch
  5. Vue.prototype
  6. patch
  7. createPatchFunction

在Vue初始化过程中，调用_init方法末尾会调用vm.$mount方法，而$mount我们之前应该讲过，其主要步骤就是为了返回内部mountComponent函数执行的值。mountComponent声明在lifecycle.js文件中，其内部定义了一个updateComponent函数，用于生成一个Watcher实例，这样数据改变触发Watcher时，则会调用updateComponent函数，即执行它内部的update方法。

之后开始vnode节点间的比较。上述中已经提及到新旧vnode分别定义了start/end两个索引，并同时向中间点逐步移动。在while循环遍历中，通过if-else判断得知，oldStartIdx/oldEndIdx会分别与newStartIdx/newEndIdx进行比较，也就是有4种可能（下列采用’==’来表示新旧两个vnode节点作sameVnode的比较），

  1. oldStartVnode == newStartVnode，则首节点相同，当前oldVnode不需要移动，StartIdx移动进入循环
  2. oldEndVnode == newEndVnode，则末节点相同，当前oldVnode不需要移动，EndIdx移动进入循环
  3. oldStartVnode == newEndVnode，此时说明原oldStartVnode已经被移动到末尾，因此将oldStartVnode移动到oldEndIdx后面位置
  4. oldEndVnode == newStartVnode，此时说明原oldStartVnode已经被移动到头部，因此将oldEndVnode移动到oldStartIdx前面位置
  5. 以上4种情况不都匹配的情况下，如果此时的vnode是有key的（v-for过程中就有要求使用key），则会根据oldVnode生成的KeyToOldIdx表去寻找newStartVnode是否存在，存在的情况下将oldVnode中的该节点移动到oldStartIdx前，如果不存在其中，则直接在对应位置新生成该DOM节点

while循环是以oldStartIdx <= oldEndIdx && newStartIdx <= newEndIdx为条件的，也就是在新旧vnode的遍历过程中，任一两个start/end不成立则退出循环，此时对于为遍历到的节点有以下处理方式，

当old先结束的情况下，若此时new中还有未遍历到的，说明是新增节点，此时获取refElm，而后将剩余的vnode全部插入到refElm前。

如果是new先结束的情况，则说明原old节点中有部分已经删除，执行removeVnodes删除多余。
